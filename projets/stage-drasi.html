Stage — DSDEN du Morbihan · Dossier E5 En cours

Site vitrine DRASI

Site web institutionnel de présentation de la DRASI (Direction des Ressources et Appui aux Systèmes d'Information) du Morbihan — équipe IT de la DSDEN sous l'académie de Rennes.

HTML5 / CSS3 JavaScript PHP Leaflet.js Git / GitHub

Contexte

Développé lors d'un stage de 6 semaines à la DSDEN du Morbihan, ce site a été réalisé en binôme (deux stagiaires BTS SIO) sous la supervision du Responsable DSI. Le projet constitue le Dossier E5 du BTS SIO SLAM.

Objectif : créer un site vitrine 100% statique (HTML/CSS/JS vanilla + PHP pour le formulaire) présentant l'équipe, les missions, le périmètre d'intervention et les services de la DRASI. Conformité obligatoire avec la charte graphique de l'Académie de Rennes.

Pages réalisées

  • Accueil — présentation générale de la DRASI
  • Notre histoire — historique du service (ex-DSII)
  • Nos équipes — membres et rôles
  • Nos missions — périmètre d'intervention
  • Notre périmètre — carte interactive (Leaflet.js)
  • Services opérés — liste des services gérés
  • Comitologie — gouvernance
  • Nous contacter — formulaire sécurisé (reCAPTCHA v2 + PHP)

Points techniques notables

  • Architecture composants : header, footer et cookie banner chargés via fetch() pour éviter la duplication
  • Carte interactive du périmètre géographique avec Leaflet.js
  • Formulaire de contact sécurisé : reCAPTCHA v2, validation PHP côté serveur, envoi par mail()
  • Système d'analytics maison (RGPD-compliant) : pages visitées, temps passé, sources — stockage localStorage, export CSV
  • Conformité charte graphique Académie de Rennes (couleurs, typos, logo Marianne)
  • Tests multi-navigateurs : Chrome, Firefox, Edge, Safari — audit Lighthouse
  • Déploiement Apache 2.4 Linux en production, versionné sur GitHub
  • Accessibilité : balises sémantiques, contrastes WCAG, navigation clavier

Sécurité

  • HTTPS avec certificat SSL, cookies httpOnly + SameSite
  • Headers Apache : X-Frame-Options, X-XSS-Protection, Content-Security-Policy
  • reCAPTCHA v2 sur le formulaire de contact
  • Validation double : client (HTML5) + serveur (PHP)
